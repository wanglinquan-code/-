购物商城系统项目计划书
一、大作业项目题目
购物商城系统
二、大作业目的
随着电子商务行业的蓬勃发展，传统线下购物模式逐渐难以满足消费者便捷化、多元化的购物需求，同时商家也亟需高效的线上管理解决方案。本购物商城系统旨在打造一款前后端分离的数字化电商平台，实现用户自助购物、商品管理、订单处理、权限控制等核心功能，既满足毕业设计对软件开发流程、技术应用能力的考核要求，又帮助开发者掌握电商平台的核心业务逻辑与架构设计思路，最终提升用户购物体验与商家管理效率。

四、项目内容
① 实验项目功能：
用户管理：支持用户注册、登录、个人信息查询与修改，通过 JWT 实现身份认证与权限控制，保障用户数据安全。
商品浏览与检索：以图文并茂的形式展示全品类商品，支持按类别分类浏览、关键词搜索，方便用户快速定位心仪商品。
商品详情查看：点击商品可查看详细信息，包括商品名称、规格、价格、库存、描述、用户评价等，为用户购买决策提供参考。
购物车管理：用户可自由选择商品数量、加入购物车，支持随时修改商品数量、删除商品，实时统计购物车总价格。
订单查询：用户确认购物车商品后可提交订单，支持订单生成、历史订单追溯等功能。
后台管理：管理员端支持商品上下架、库存调整、订单状态处理（接单、发货、退款）、用户权限管理等操作，实现商家高效运营。
② 项目步骤以及进度表
需求分析与架构设计：
完成第 1-2 天、查阅电商行业相关资料与同类平台案例，明确用户端与管理端的核心需求。撰写详细的需求规格说明书，界定功能需求（用户注册登录、商品检索、订单管理等）与非功能需求（系统响应速度、操作便捷性、数据安全性）。基于前后端分离架构理念，设计系统整体架构，明确前端页面结构、后端接口规范、数据库表设计及缓存策略。
界面设计：
完成第 3 天、根据需求规格说明书及电商平台用户体验规范，进行前后端界面原型设计。设计包括用户端的商品浏览页、商品详情页、购物车页、订单页，以及管理端的商品管理页、订单处理页、用户管理页等的布局、色彩搭配、图标设计及交互流程，确保界面简洁直观、操作流畅。
前后端开发：
完成第 4-7 天、前端搭建页面组件，实现商品展示、搜索、购物车交互、订单提交等用户端功能；后端基于 Spring Boot 框架开发接口，实现数据逻辑处理、数据库交互（MySQL）、高频数据缓存（Redis）、用户认证（JWT）等核心功能；前后端同步开发，确保接口适配与功能联动。
联调测试：
完成第 8 天、进行前后端联调，验证接口通信稳定性与功能完整性。开展全面测试，包括功能测试（验证各模块功能正常运行）、性能测试（确保系统响应快速）、数据安全测试（保障用户信息与交易数据安全），及时发现并修复 bug。
优化完善与文档撰写：
完成第 9 天、根据测试结果优化系统性能，提升操作便捷性与页面加载速度。撰写完整的项目文档，包括需求规格说明书、设计文档、源码注释、测试报告等，整理可运行的系统源码。
答辩演示准备：
完成第 10 天、整理项目演示流程，准备演示用例，调试系统运行环境，确保答辩时能够清晰展示系统核心功能、技术亮点及开发成果。
